# Обзор API

## Общий принцип

Основным публичным API сервиса считается `/api/v1`.

Маршрут `/api/bootstrap` сохранен как legacy-интерфейс для совместимости с текущим frontend и не должен рассматриваться как основной внешний API для интеграций.

Полная интерактивная документация доступна через Swagger UI:
- `http://localhost:3001/docs`

## Основные endpoint'ы

### Проверка доступности

- `GET /api/v1/health`  
  Проверка доступности backend API и подключения к базе данных.

### Студенты

- `GET /api/v1/students`  
  Получение списка студентов.

Поддерживаемые query-параметры:
- `search`
- `program`
- `status`
- `language`
- `skill`
- `sortBy`
- `sortDir`
- `page`
- `pageSize`

- `GET /api/v1/students/{id}`  
  Получение полной карточки одного студента.

- `POST /api/v1/students`  
  Создание студента.

- `PUT /api/v1/students/{id}`  
  Обновление данных студента.

- `DELETE /api/v1/students/{id}`  
  Удаление студента.

### Согласия

- `GET /api/v1/consents`  
  Получение списка согласий.

- `POST /api/v1/consents`  
  Создание согласия.

- `PUT /api/v1/consents/{id}`  
  Обновление согласия.

### Справочники

- `GET /api/v1/programs`  
  Получение списка образовательных программ.

- `GET /api/v1/languages`  
  Получение списка языков.

- `GET /api/v1/skills`  
  Получение списка hard skills.

- `GET /api/v1/soft-skills`  
  Получение списка soft skills.

## Формат ответов

Для основных `GET`-эндпоинтов используется единый формат:

```json
{
  "data": [],
  "meta": {
    "total": 0
  }
}
```

Для получения одной сущности:

```json
{
  "data": {
    "id": "student-1"
  }
}
```

## Формат ошибок

Ошибки отдаются в следующем формате:

```json
{
  "error": {
    "code": "student_not_found",
    "message": "Student not found"
  }
}
```

Примеры типовых статусов:
- `400` - ошибка валидации
- `404` - сущность не найдена
- `409` - конфликт уникальных данных
- `500` - внутренняя ошибка сервера
